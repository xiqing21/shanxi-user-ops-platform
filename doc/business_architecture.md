# 国家电网配变与线路状态实时监测平台——业务架构设计说明书

> **文档状态**：发布版 (v1.0.0)  
> **面向对象**：电力业务分析师、算法工程师、Flink 开发人员  
> **核心目标**：统一电网监控指标定义，明确配变与线路三载、复杂事件（CEP）及数据订正业务口径。

---

## 一、 核心业务对象与实体

在配电网状态监测业务中，存在以下核心实体及其拓扑关系：

```
[变电站 (Substation)]
       │ (1 : N)
       ▼
[中压配电线路 (Feeders/10kV Lines)]
       │ (1 : N)
       ▼
[配电变压器 (Distribution Transformers / 配变)]
       │ (1 : N)
       ▼
[低压用户线路/智能表计 (Smart Meters / 380V/220V)]
```

### 实体元数据定义

| 业务实体 | 核心属性 | 数据来源系统 | 业务说明 |
| :--- | :--- | :--- | :--- |
| **变电站** | 站ID、站名称、电压等级、地理坐标 | GIS系统 / 调度系统 | 配电网的电源节点 |
| **配电线路** | 线路ID、线路名称、额定电流、导线型号、所属变电站 | GIS系统 / 营销系统 | 输送电能的物理载体，存在重过载风险 |
| **配电变压器** | 配变ID、容量(kVA)、变比、所属线路、安装位置 | PMS系统 (生产管理) | 将高压降为民用低压的核心设备 |
| **智能电表** | 电表ID、用户类型、所属配变、安装位置 | 采集系统 (营销) | 采集末端电压、电流、电量数据 |

## 二、 遥测业务指标与三载定义

### 1. 核心遥测指标表 (Telemetry Metrics)
智能电表与线路传感器每 5-15 秒上报一次数据包，包含以下关键遥测指标：

$$I_{load\_rate} = \frac{P_{active}}{S_{capacity} \times \cos\phi} \times 100\%$$

*   \(P_{active}\)：当前有功功率 (kW)
*   \(S_{capacity}\)：变压器额定容量 (kVA)
*   \(\cos\phi\)：功率因数 (一般取 0.9)

以及：
*   \(U_A, U_B, U_C\)：三相电压 (V)
*   \(I_A, I_B, I_C\)：三相电流 (A)
*   \(T_{line}\)：线路/电缆头运行温度 (°C)

---

### 2. "三载" (三级负荷) 状态分类标准

配变与线路在不同时间段的负载率决定了其安全等级，具体业务分类如下：

| 负载状态 | 配变负载率 (\(I_{load\_rate}\)) | 线路负载率 (\(I_{line\_rate}\)) | 业务定义与影响 |
| :--- | :--- | :--- | :--- |
| **轻载 (Light)** | \(< 20\%\) | \(< 15\%\) | 资源闲置，代表设备选型过大或区域用电量低。 |
| **合理 (Reasonable)** | \(20\% \le I \le 80\%\) | \(15\% \le I \le 70\%\) | 经济运行区间，设备损耗低，寿命最长。 |
| **重载 (Heavy)** | \(80\% < I \le 100\%\) | \(70\% < I \le 90\%\) | 临界警告区间，设备损耗加剧，需限制新增用电接入。 |
| **过载 (Overload)** | \(> 100\%\) | \(> 90\%\) | **严重安全隐患**，长时间过载会导致绝缘老化甚至烧毁变压器。 |

## 三、 Flink CEP 复杂事件告警场景

电力系统的异常往往具有时序链式特征，单一的数据点异常不代表设备故障。必须通过复杂事件处理 (CEP) 进行多点结合判定：

### 场景 1：重载向过载渐进性恶化预警
*   **业务描述**：当某台配电变压器负载率在过去 15 分钟内，经历“合理 -> 重载 -> 过载”的连续上升趋势，且持续时间超过 5 分钟，说明该变压器负荷陡增，极可能因过热烧毁。
*   **CEP 匹配模式**：
    1.  事件 A：负载率处于 Reasonable (\(20\% \sim 80\%\))；
    2.  紧接着事件 B：负载率进入 Heavy (\(80\% \sim 100\%\)) 并持续至少 5 分钟；
    3.  紧接着事件 C：负载率突破 Overload (\(> 100\%\))。

### 场景 2：线路温度陡增与高负荷耦合预警
*   **业务描述**：输电线路在用电负荷增加的同时，线温在 1 分钟内急剧升高超 5°C，或温度直接突破 85°C 阈值。
*   **CEP 匹配模式**：
    1.  事件 A：线路负荷大于 70% 且温度平稳；
    2.  事件 B：在一分钟（时间窗口）内，连续发生线温差值 \(\Delta T \ge 5^\circ\text{C}\)。

### 场景 3：三相不平衡度持续扩大告警
*   **业务描述**：三相负载不平衡会导致中性线电流过大，变压器局部过热。当三相电流不平衡度 \(I_{unbalance} > 30\%\) 持续时间超 30 分钟，触发告警。
*   **计算公式**：
    $$I_{unbalance} = \frac{I_{max} - I_{min}}{I_{max}} \times 100\%$$

## 四、 历史变更与删除 Retraction 业务场景

在生产环境中，数据不全是单向追加的，以下场景会产生撤回/更新（Retraction）流：

### 1. 遥测数据回补与订正 (Retroactive Meter Adjustments)
*   **业务场景**：某智能表计由于通信基站故障，在 14:00 - 14:30 期间上报了错误的超高负载电流数据（例如 1000A，实际仅 100A），触发了过载 CEP 告警。15:00 通信恢复后，采集系统通过离线通道重新补发了该时段的正确历史表计数据。
*   **业务要求**：系统必须撤回 14:00 - 14:30 产生的错误过载告警，并在 StarRocks 大屏统计数据中自动减去异常电量，重新合并为最新的正确状态。

### 2. 物理设备解绑与删除 (Equipment Deletion)
*   **业务场景**：某过载台区因电网改造被拆除，或某表计从变压器上解绑并报废。营销系统发送了 `DELETE` 变更消息。
*   **业务要求**：DWD 宽表必须撤回该设备的所有实时聚合值，且大屏实时大图拓扑中要实时移除该设备节点，不能残留僵尸指标。

### 3. Flink CEP 面对 Retraction 流的局限性与突破
*   **痛点**：Flink CEP 仅支持 **Append-Only（仅追加）** 的输入流。当 Flink CDC / Fluss 输入源中混入 `DELETE`（删除）或 `UPDATE_BEFORE / UPDATE_AFTER`（更新）操作时，Flink CEP 算子会直接报错抛出异常。
*   **业务应对策略**：在进入 CEP 告警算子前，必须通过数据清洗过滤，对更新流执行“只保留最终态/折中插入（Insert-Only 转换）”，或将 `DELETE` 转换为带有删除标志位的插入数据，避免 Retraction 流引起 CEP 引擎崩溃。
